# Weekly Trend Report

Generate a weekly rollup analyzing trends across the last 7 days.

## Step 1: Gather Data

1. Read `config.yaml` for Obsidian vault path
2. Read all daily notes from `{vault_path}/{daily_folder}/` for the last 7 days
3. Run the scoring engine with weekly flag:
```bash
cd <project_root> && .venv/bin/python -m src.scoring --weekly
```
This outputs: persistent topics (5+ days), flash topics (peaked and faded), rising topics, and top 5 signals.

## Step 2: Analyze Weekly Patterns

Using the daily notes + scoring output, identify:

- **Persistent Trends** (appeared 5+ of 7 days): These are the real trends, not noise
- **Flash Signals** (1-2 days, high score, then gone): What hyped up and disappeared?
- **Rising Signals** (growing each day): What to watch next week
- **Best Ideas**: Rank all ideas from the week's idea notes by signal score
- **Source reliability**: Which sources produced the most actionable signals?
- **Newsletter accuracy**: Did newsletter-mentioned topics become bigger trends?

## Step 3: Write Weekly Note

Write to: `{vault_path}/{weekly_folder}/{year}-W{week_number}.md`

```markdown
---
date: {today}
type: weekly-digest
week: {year}-W{week_number}
tags: [weekly-digest, {top_tags}]
---

# Weekly Trend Report — {year}-W{week_number}

## Week Summary
{3-5 sentence overview: what dominated, what surprised, what to watch}

## Persistent Trends (5+ days)
### {topic}
- **Days active**: {count}/7

- **Trajectory**: rising / stable / declining
- **Daily notes**: [[{date1}]], [[{date2}]], ...
- **Business opportunity**: {if any emerged during the week}

## Flash Signals (peaked and faded)
### {topic}
- **Active**: {date_range}
- **Peak**: {score}/100 on {date}
```
