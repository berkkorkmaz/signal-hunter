# Daily Knowledge Digest

You are running the Daily Knowledge Builder pipeline. First, determine the project root (where `config.yaml` lives) and read `config.yaml` to get all settings including the Obsidian vault path.

## Step 1: Run Python Collector (Reddit, YouTube, X/Twitter, App Stores)

Run the collector script from the project root:

```bash
cd <project_root> && .venv/bin/python -m src.collector
```

Save the JSON output — it contains Reddit top posts, YouTube videos with transcripts, X tweets, and app store data. The collector uses daily caching so re-runs are free.

## Step 2: Fetch Web Sources via WebFetch

Read `config.yaml` → `sources.webfetch` for the list of URLs. For each, use WebFetch to extract content:

- Extract titles, URLs, scores/points, descriptions
- Fetch in parallel where possible

## Step 3: Fetch Email Newsletters via Gmail MCP

Check `config.yaml` → `sources.gmail`. If `auto_discover: true`:
```
gmail_search_messages with q: "unsubscribe" newer_than:1d category:updates
```

For each newsletter (filter out promotional/transactional emails):
- Read with `gmail_read_message`
- Extract key AI/tech/business content

## Step 4: Read Yesterday's Note (for novelty detection)

Read the Obsidian vault path from `config.yaml` → `obsidian.vault_path`.
Check if yesterday's daily note exists at `{vault_path}/{daily_folder}/{yesterday_date}.md`.
If it exists, read it to identify what topics are NEW today vs continuing trends.

## Step 5: Synthesize & Analyze

Combine ALL collected data and perform:

### a) Cross-Source Trend Detection
Find topics, projects, companies, or technologies appearing in **3+ different sources**.
Rate each signal 1-10 based on:
- Number of distinct sources mentioning it
- Total engagement (upvotes, stars, comments)
- Novelty (is this the first day it appears?)

### b) Business Idea Extraction
For each strong trend, brainstorm actionable ideas:
- **App ideas** (mobile/web)
- **SaaS opportunities**
- **BaaS (Backend as a Service)** plays
- **Gaming applications**
Focus on market gaps and problems that need solving.

### c) Executive Summary
Write 3-5 sentences covering the day's most important developments.

## Step 6: Write Obsidian Daily Note

Read paths from `config.yaml` → `obsidian` section. Write to:
`{vault_path}/{daily_folder}/{today_date}.md`

Use this format:

```markdown
---
date: {today_date}
type: daily-digest
tags: [daily-digest, {top_tags}]
sources_ok: [{list of successful sources}]
sources_failed: [{list of failed sources}]
---

# Daily Knowledge Digest — {today_date}

## Executive Summary
{3-5 sentence overview}

## Trending Topics (Cross-Source Signals)
### {topic_name}
- **Signal Strength**: {score}/10
- **Seen in**: {source1}, {source2}, {source3}...
- **Summary**: {what's happening}
- **Why it matters**: {analysis}
- **Business angle**: {opportunity if any}

## Business Ideas & Opportunities
### {idea_title}
- **Category**: app / saas / baas / gaming
- **Inspired by**: {source items}
- **Market gap**: {what's missing}
- **Difficulty**: low / medium / high
- **Next step**: {what to do first}

---

## Source Details
{Per-source sections with titles, URLs, scores}

## Source Status
| Source | Status | Items |
|--------|--------|-------|
{status table}

---
*Generated by Daily Knowledge Builder*
```

For each strong business idea (signal strength >= 7), create a separate note:
`{vault_path}/{ideas_folder}/idea-{today_date}-{slug}.md`

```markdown

type: idea
```
